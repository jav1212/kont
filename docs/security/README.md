# Security Docs

This folder documents the current security model and the observed security posture of `Kont`.

## Purpose

Because the system is evolving quickly, security documentation must make clear:

- what security controls currently exist
- where the strongest protections are
- where the current weaknesses or risks are
- what should be hardened next

## Scope

These documents are based on repository inspection and current architecture review.

They are:

- a documentation and architecture assessment
- a security posture review

They are **not**:

- a formal pentest
- a red-team exercise
- a cryptographic review
- a guarantee that the system is secure
